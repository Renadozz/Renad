# Enemy Skills and AI Patterns

## Combat Implementation Rules

| Rule                    | Implementation                                                                                                                        |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Physical Damage         | Uses the attacker's `Physical Attack` against the player's `Defense`. Physical skills perform an Accuracy vs. Evasion check.          |
| Magic Damage            | Uses the attacker's `Magic Power` against the player's `Magical Defense`. Magic skills always hit.                                    |
| Element                 | An element is applied in addition to the damage type. Example: `Fire Bolt` is Magic Damage with the Fire element.                     |
| Primary Element         | A skill with `Primary` uses the first element listed on the enemy. A Neutral enemy deals Neutral damage.                              |
| `P100`                  | 100% of the normal Physical Damage formula before Defense reduction.                                                                  |
| `M100`                  | 100% of the normal Magic Damage formula before Magical Defense reduction.                                                             |
| Status Chance           | Status chances are checked after a successful hit. Status Resistance reduces the final chance.                                        |
| Status Reapplication    | Do not apply the same negative status while it still has at least 1 turn remaining. Skip that AI rule and evaluate the next priority. |
| Buff Reapplication      | Do not apply the same buff while it still has at least 1 turn remaining. Skip that AI rule and evaluate the next priority.            |
| Cooldown                | After a skill is used, set its cooldown to the listed value. Reduce all cooldowns by 1 at the start of that enemy's next turns.       |
| Maximum Enemies         | An encounter can contain a maximum of 3 living enemies. Summons count toward this limit.                                              |
| Boss Summons            | A boss can only use `Summon Minion` when fewer than 3 enemies are alive. Each boss can summon a maximum of 2 times per battle.        |
| AI Evaluation           | Evaluate AI priorities from top to bottom. The first usable action with valid conditions and a valid target is used.                  |
| Fallback                | If no priority is valid, use `Basic Attack` against the player.                                                                       |
| Magic-Focused Player    | The player is Magic-Focused when `Magic Power > Physical Attack`.                                                                     |
| Physical-Focused Player | The player is Physical-Focused when `Physical Attack >= Magic Power`.                                                                 |

## Targeting Terms

| Term                       | Meaning                                                                                                                   |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Player                     | The single player character. All enemy attacks and debuffs against the player always affect exactly 1 target.             |
| Self                       | The enemy using the skill.                                                                                                |
| Friendly                   | Any living enemy in the current encounter, including the caster.                                                          |
| Ally                       | Any living enemy in the current encounter except the caster.                                                              |
| Lowest HP Friendly         | The living friendly target with the lowest current HP percentage.                                                         |
| Highest Physical Friendly  | The living friendly target with the highest Physical Attack.                                                              |
| Highest Magic Friendly     | The living friendly target with the highest Magic Power.                                                                  |
| Friendly With Most Debuffs | The living friendly target with the highest number of negative effects. If tied, choose the one with lower HP percentage. |
| All Friendlies             | Every living enemy in the encounter, including the caster. Maximum 3 targets.                                             |
| No Target                  | Used by summon skills. The skill creates a new enemy instead of selecting a target.                                       |

> Target selection is defined only in the AI Priority tables. The skill library only defines which target categories are allowed.

## Default Status Durations

| Effect Type                                                                                                                            | Default Duration |
| -------------------------------------------------------------------------------------------------------------------------------------- | ---------------: |
| Burn, Poison, Bleeding, Curse                                                                                                          |          3 turns |
| Weakness, Magic Break, Armor Break, Magic Vulnerability, Blind, Slow, Crippled, Wet, Scorched, Cracked, Winded, Shadow Mark, Holy Mark |          2 turns |
| Stun, Sleep, Freeze, Confusion, Fear, Silence                                                                                          |           1 turn |
| Fortify, Barrier, Haste, Focus, Evasion Up, Magic Ward                                                                                 |          2 turns |
| Regeneration, Berserk                                                                                                                  |          3 turns |

## AI Priority Syntax

| Syntax            | Meaning                                                                             |
| ----------------- | ----------------------------------------------------------------------------------- |
| `Opening`         | Only valid on the enemy's first active turn.                                        |
| `Group`           | Use this priority list when at least one other friendly enemy is alive.             |
| `Solo`            | Use this priority list when the enemy is the only living enemy.                     |
| `HP <= X%`        | Only valid when the caster's current HP percentage is at or below the listed value. |
| `Target HP <= X%` | Only valid if a selected friendly target is at or below the listed HP percentage.   |
| `No [Effect]`     | Only valid if the selected target does not currently have that effect.              |
| `→ [Target]`      | Defines the exact target selection for this specific AI rule.                       |
| `Fallback`        | Use this action if no earlier priority is valid.                                    |

### AI Priority Example

```md
1. Opening: Haste → [Highest Physical Friendly without Haste]
2. Bleeding Attack → [Player if no Bleeding]
3. Barrier → [Lowest HP Friendly at or below 55% without Barrier]
4. Basic Attack → [Player]
```
